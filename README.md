# AI Dungeon Master

A text-adventure dungeon master powered by Claude. Type what you do; the world responds with streaming second-person narration, illustrated scenes, a persistent character, a living world full of NPCs, and a hidden 3-act story arc that adapts to your choices.

Built on Next.js 16 (App Router, Turbopack) · TypeScript · Tailwind v4 · Anthropic Claude (Haiku 4.5) · Together AI (FLUX-schnell) · Supabase (Postgres + Storage).

> **Live demo:** *(add Vercel URL once deployed)*

## What it does

You type an action. Claude streams a few paragraphs of vivid narration. Behind the scenes it's calling a fleet of structured tools — rolling dice, taking your HP, putting items in your pack, recording the NPCs you've met, advancing the in-world clock, generating a scene illustration through FLUX, and quietly pushing you through the next beat of a hidden 3-act arc that was generated when you started the campaign.

Your character sheet, world state, quest log, and a per-character portrait that adapts to your gear all live in a left sidebar. Everything persists across reloads.

## Features

### Narration
- **Streaming SSE turn loop** with rAF-batched client rendering — markdown re-renders once per frame, never per token.
- **Anthropic tool use** (18 tools): `roll_dice`, `update_hp`, `add_item` / `remove_item`, `apply_status_effect`, `record_npc`, `record_location`, `record_lore`, `record_quest`, `update_quest_status`, `start_encounter`, `damage_enemy`, `defeat_enemy`, `end_encounter`, `advance_time`, `set_scene`, `advance_arc`, …
- **Optimistic UI** — tool events stream over the SSE channel and mutate sidebar state the moment the DM invokes them, before the network round-trip to Supabase finishes.
- **Prompt caching with a cache-split system prompt** — static instructions are cached; dynamic state (HP, inventory, world) is sent uncached, giving ~90% input-token discount on every turn after the first.
- **Sliding-window summarization** — old turns get rolled into a single summary block once the conversation exceeds a budget, so cost-per-turn stays constant no matter how long the campaign runs.

### Character
- Five starting classes (Wanderer, Fighter, Rogue, Mage, Ranger) with preset HP, attributes, skills, and starting gear.
- **HP bar that ramps smoothly through HSL** from green (full) to amber to red (low), pulsing under 30%.
- **Generative character portraits** — FLUX-schnell builds a head-and-shoulders portrait from class + currently-carried gear + visible status effects. Signature-hashed so identical loadouts share an image; new gear triggers a fresh portrait in the background.
- Status effects (buffs, debuffs, conditions, injuries) with expiry timers tied to in-world minutes.

### World
- **NPCs** with attitude (`friendly` / `hostile` / `suspicious` / `allied` / `fearful` / `neutral`) and a relationship score the DM can update.
- **Locations** and **lore** the DM records as you discover them — surfaced in a collapsible world panel.
- **Clock + weather** that the DM advances in minutes; days roll over automatically.
- **Encounters** — distinct mode the DM enters with `start_encounter`, tracked enemies with HP bars, ends with `end_encounter`.
- **Quests** with `active` / `completed` / `failed` / `abandoned` status.

### Story structure
- **Hidden 3-act arc** generated by a separate Claude call at campaign creation, stored as DM-private notes injected into the prompt.
- The DM is told its current and next beat goals as hidden notes, so narrative pressure increases at the right moments without the player seeing the scaffolding.
- **Fork-from-here** — clone a campaign at any prior message into a fresh save-point branch, including character state, world, statuses, and quests.

### Sensory
- **Scene illustrations** via Together AI FLUX-schnell (~1–3s per image), cached permanently in Supabase Storage by prompt hash.
- **Cinematic mode** (default) — full-width scene image is primary, chat collapses to a toggle.
- **Browser TTS** with voice picking + `voiceschanged` event handling.
- **Procedural ambient audio** generated in Web Audio (brown-noise + biquad + LFO), with 5 mood configs and crossfade.
- **Typewriter fade** on streaming text.

### Operational polish
- **Per-visitor isolation** via a `visitor_id` cookie set by a Next.js proxy on first request. No login; everyone gets their own campaigns.
- **Share links** — read-only public URL for any campaign.
- **Live cost tracking** — running total of Anthropic spend per campaign, with cache-hit-rate breakdown.
- **In-memory rate limits** (40 turns/hr, 12 new adventures/hr per visitor) to keep a public deploy from getting bot-drained.

## Architecture highlights

A few pieces that I think are interesting beyond "AI app glued together":

**The cache split.** Anthropic's prompt cache requires byte-identical prefixes. Mixing dynamic state (HP changes every turn) into the system prompt would invalidate the cache every turn. So the system prompt has two blocks: a large static instructions block with `cache_control: ephemeral`, and a small dynamic block (current HP, world clock, recent statuses) appended without cache marking. Every turn after the first reads the static prefix from cache at ~10% of the cost.

**Tool execution with optimistic UI.** When the DM calls `add_item`, the server emits a `tool_event` over SSE *before* the Supabase write returns, so the inventory pill shows up in the sidebar instantly. The actual DB mutation runs in parallel; on stream end the client refetches authoritative state.

**Portrait signature hashing.** Regenerating a portrait every turn would burn ~$0.003 per turn pointlessly. Instead, a stable signature hash of `class + name + sorted inventory + sorted statuses` is computed; only when that hash changes does FLUX get called. The hash also doubles as the storage path, so identical signatures share a single cached image.

**Sliding-window summarization with a cursor.** A `summary_through_message_id` column tracks how far the summarized state extends. New turns from beyond that cursor are included verbatim; older ones are folded into a single summary block via a separate Claude call when the window grows past a threshold. Keeps token cost flat for arbitrarily long campaigns.

**Background-fired heavy work.** Portrait regen, story-arc generation, and scene illustration all fire as un-awaited promises from the request path, so the SSE stream never blocks on a 2–3s FLUX call. Clients pick up the results via polling or the next-turn refetch.

## Tech stack rationale

- **Claude Haiku 4.5** for narration + tool calls — fast enough for streaming, ~$0.0003 per turn at typical context, vivid prose. Sonnet would be richer but ~10× the cost.
- **Together AI FLUX-schnell** for images. ~$0.003 per image at 768×432 in 1–3s, vs ~60–90s on free Pollinations. Worth the cost.
- **Supabase** for Postgres (campaigns, characters, world tables) + Storage (cached portraits + scenes). Service-role admin key on the server, anon key in the browser.
- **Tailwind v4** with `var(--accent)` driving the warm-amber palette throughout.
- **rAF-batched streaming reader** — the same pattern from my Paper Summarizer project: each token push only queues a re-render on the next animation frame, never per token.

## Run locally

```bash
git clone https://github.com/henrymyos/ai-dungeon-master.git
cd ai-dungeon-master

cp .env.example .env.local
# Fill in keys (see below)

npm install
npm run dev
# → http://localhost:3000
```

### Required environment variables

```bash
ANTHROPIC_API_KEY=             # console.anthropic.com/settings/keys
NEXT_PUBLIC_SUPABASE_URL=      # any Supabase project URL
NEXT_PUBLIC_SUPABASE_ANON_KEY= # safe to expose
SUPABASE_SERVICE_ROLE_KEY=     # server-only, never expose
TOGETHER_API_KEY=              # api.together.xyz/settings/api-keys (optional — falls back to Pollinations)
```

### Database schema

Tables are `dm_campaigns`, `dm_messages`, `dm_characters`, `dm_npcs`, `dm_locations`, `dm_lore`, `dm_statuses`, `dm_encounters`, `dm_enemies`, `dm_quests`. Storage buckets are `scenes` and `portraits`, both public.

Schema is managed via Supabase migrations applied through the MCP tool during development. The migration files aren't versioned in this repo yet — running this against a fresh Supabase project would require recreating the schema by hand.

## Deploy

The app is designed for Vercel:

1. Push to GitHub (this repo).
2. Import into Vercel — Next.js is auto-detected.
3. Add the five environment variables above in **Project Settings → Environment Variables**.
4. Deploy.

**Before going public:** set spending caps on all three provider dashboards (Anthropic, Together AI, Supabase). The in-memory rate limit is best-effort — across Vercel's serverless instance pool it leaks a bit, so it's not a substitute for hard provider-side caps.

## Limitations

- **No auth.** Identity is a visitor cookie. Clearing cookies or switching browsers means starting over.
- **Schema isn't versioned** as migration files in this repo — it lives in Supabase. Fresh deployments need it recreated.
- **In-memory rate limit doesn't survive cold starts.** Real production protection would need Supabase- or Redis-backed counters.
- **No character continuity across portraits.** FLUX is stateless, so a class change generates a brand-new face rather than the same face wearing different gear.
